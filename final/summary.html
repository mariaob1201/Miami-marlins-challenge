Hierarchical Bayesian Exit Velocity Talent Model¶

Miami Marlins Challenge - Player Performance Analytics
Author: Maria Oros
Date: September 21, 2025

Executive Summary¶

This analysis develops a sophisticated hierarchical Bayesian framework to evaluate batter exit velocity talent, separating true underlying ability from noise and contextual factors. The model processes data from 3,715 batters across MLB, AAA, and AA levels.

Key Findings¶

  • League Equivalencies: AAA performance translates to approximately 2.5 mph lower exit velocity than MLB, while AA shows a 4.5 mph gap
  • Age Effects: Peak performance occurs around age 27-28, with quadratic decline thereafter
  • Shrinkage Benefits: Players with limited data show 40-60% shrinkage toward population mean, improving projection accuracy
  • Uncertainty Quantification: Model provides credible intervals reflecting confidence in each projection

Forecasting Modeling: ARIMA Time Series Approach¶

The source code for the ARIMA modeling approach can be find here 3_Modeling/time_series/arima.py

Model overview¶

For each batter, let ( y_t ) be the seasonal mean exit velocity (EV) in year ( t ). Let ( \mathbf{x}_t \in [\mathbf{x}_t \in \mathbb{R}^{3} ]) be the league-share vector ordered as [ \text{AA},\ \text{AAA},\ \text{MLB} ] (components sum to 1 when the player has data that year). We fit an ARIMAX model per batter:

ARIMAX(p,d,q) with exogenous level shares

[ \phi(B)\,(1 - B)^d\, y_t \;=\; c \;+\; \boldsymbol{\beta}^\top \mathbf{x}_t \;+\; \theta(B)\,\varepsilon_t ]

where: - ( B ) is the backshift operator ( (B y_t = y_{t-1}) ) - ( \phi(B) = 1 - \phi_1 B - \cdots - \phi_p B^p ) (autoregressive) - ( \theta(B) = 1 + \theta_1 B + \cdots + \theta_q B^q ) (moving-average) - ( d \in {0,1} ) (minimal differencing due to short annual histories) - ( \varepsilon_t \sim \mathcal{N}(0, \sigma^2) )

We restrict ( (p,d,q) \in {0,1}^3 ) to avoid overfitting short series.

Exogenous regressors (league effects)¶

The coefficient vector ( \boldsymbol{\beta} ) captures expected EV shifts by level. To forecast a scenario where the player is at a particular level in 2024, set ( \mathbf{x}{2024} ) accordingly (e.g., MLB scenario: ( \mathbf{x}{2024} = [0,\,0,\,1] )).

Data preparation¶

  • Aggregate batted-ball rows to batter × season: ( y_t = \mathrm{mean}(\text{exit_velo}) ).
  • If a batter plays multiple levels in a season, compute level shares from counts of observations.
  • Use an annual PeriodIndex for seasons to give the model a supported frequency.
  • Enforce strict exogenous column order ([ \text{AA}, \text{AAA}, \text{MLB} ]) for both fitting and forecasting.

Order selection and validation¶

  • When there are enough seasons to allow at least one rolling-origin fold ( ( \text{initial_window} + \text{horizon} < n ), with ( \text{horizon} = 1 ) season ), perform rolling one-step-ahead cross-validation:
  • For ( t = \text{initial_window}, \ldots, T-1 ), fit on seasons up to ( t ) and predict ( t+1 ).
  • Compute RMSE across the one-step forecasts.
  • Choose ( (p,d,q) ) with the lowest RMSE (tie-break by AIC).
  • If cross-validation is not possible (very short histories), select ( (p,d,q) ) by full-sample AIC.

Forecasting 2024¶

  • Refit ARIMAX on all available seasons with the chosen ( (p,d,q) ).
  • Construct ( \mathbf{x}{2024} ) for the desired level scenario and generate a one-step-ahead forecast ( y{T+1} ).
  • Report a 95% prediction interval using the model’s forecast standard error (approximate normality).

Fallback rule (transparent)¶

If order selection or refitting fails, or the series is too short, use a simple baseline:

[ \hat{y}{T+1} \;=\; y_T \;+\; \Delta{\text{level}}(\text{target}) \;-\; \Delta_{\text{level}}(\text{last}) ]

with fixed level adjustments (league equivalencies): - ( \Delta_{\text{level}}(\text{MLB}) = 0.0 ) mph
- ( \Delta_{\text{level}}(\text{AAA}) = -2.5 ) mph
- ( \Delta_{\text{level}}(\text{AA}) = -4.5 ) mph

These offsets can be recalibrated from league-wide data.

Assumptions and limitations¶

  • Linearity in level shares ( \boldsymbol{\beta}^\top \mathbf{x}_t ) and time-invariant level effects.
  • Homoskedastic Gaussian errors at the seasonal level.
  • Short histories are common; to avoid instability we restrict ( (p,d,q) ) to ( {0,1} ).
  • No additional covariates beyond league shares (by design for a fast, interpretable baseline).

Diagnostics to report¶

  • Per batter: chosen ( (p,d,q) ), cross-validated RMSE (if available), AIC, whether fallback was used.
  • Aggregate: distribution of RMSE across modeled batters; percent modeled vs. fallback; percent beating a naïve last-value baseline.
